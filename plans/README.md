# Implementation Plans

Generated by the `improve` skill on 2026-06-12, written against commit `71d6ed71`
(QuickAdd 2.13.1). Each plan is **self-contained**: an executor with zero context
from the audit session can run it from the plan file + the repo alone.

**Before executing any plan**: read it fully, run its `Drift check` first, honor
its `STOP conditions`, and update this file's status row when you finish.

**Hard constraints for every executor** (repeated from the plans):
- Package manager is **bun** — never npm/yarn/pnpm.
- **Never** run `bun run build` or `bun run dev`: they regenerate the bundled
  `main.js`/`styles.css` at the repo root, which must not be hand-edited.
  Typecheck with `bunx tsc -noEmit -skipLibCheck` instead.
- Verification gates: `bun run test`, `bunx tsc -noEmit -skipLibCheck`,
  `bun run lint`, `bun run check` (svelte-check).
- Do not push or open a PR unless explicitly instructed.

## Findings considered and rejected

- **CORRECT-06** ("Preflight VDATE scan ANDs the optional flag onto same-named
  non-date requirements") — **rejected as by-design**. `{{VALUE:x}}` and
  `{{VDATE:x}}` resolve the *same* runtime variable into one form field; the
  documented contract (`RequirementCollector.ts:38`) is variable-level, not
  per-type, and "required wins via AND across all occurrences" is the only
  coherent resolution. The proposed change would let a required VDATE occurrence
  be silently skipped and is pinned against by existing tests
  (`RequirementCollector.test.ts:263-297`). Not a bug.
